# Windows-Style Tiling on Pop!_OS (GNOME on Wayland) — Tiling Assistant

## Context
- OS: Pop!_OS 24.04 LTS
- Session: GNOME on Wayland
- Goal: Get Windows-style snap-tiling behavior (drag-to-edge layouts + auto-fill remaining space)

## What this gives you
Vanilla GNOME only supports basic drag-to-edge tiling (left/right half), with no extras. **Tiling Assistant** adds:
- Drag-to-edge and drag-to-corner tiling (halves, quarters, thirds)
- A popup after tiling a window, letting you pick another open window to auto-fill the remaining space
- Tile groups, gaps, and keyboard-shortcut behavior, all configurable

> Note: this isn't a literal "hover over maximize button" popup like Windows 11 Snap Layouts — Tiling Assistant's popup appears after you've already tiled a window, to help you fill the rest of the screen. Functionally it gets you to the same place (fast multi-window layouts) via drag-to-edge instead of a hover menu.

## Setup

### Step 1: Install GNOME Tweaks (optional but recommended)
Lets you enable window minimize/maximize titlebar buttons, which makes hover/click tiling more natural to use.

**Terminal:**
```bash
sudo apt install gnome-tweaks
```
**GUI:** Open **Pop!_Shop** (or Software Center) → search "GNOME Tweaks" → Install.

Then: Open **Tweaks** → **Window Titlebars** → enable **Minimize** and **Maximize**.

### Step 2: Install Extension Manager (if not already installed)
**Terminal:**
```bash
sudo apt install gnome-shell-extension-manager
```
**GUI:** Open **Pop!_Shop** → search "Extension Manager" → Install.

### Step 3: Install Tiling Assistant
Tiling Assistant isn't in standard apt repos, so install via the Extension Manager GUI.

**GUI:**
1. Open **Extension Manager**
2. Go to the **Browse** tab
3. Search "Tiling Assistant"
4. Click **Install**

### Step 4: Enable the extension
**GUI:** In **Extension Manager** → **Installed** tab → find **Tiling Assistant** → toggle it **ON**.

**Terminal (alternative):**
```bash
gnome-extensions enable tiling-assistant@ubuntu.com
```

### Step 5: Configure settings
Open **Extension Manager** → **Tiling Assistant** → click the ⚙️ icon to open settings, split across **General** and **Keybindings** tabs.

#### General tab

**Tiling Popup**
- **Open after tiling a window** — shows a popup listing your other open windows so you can pick one to fill the remaining screen space right after you tile a window. On by default.
- **Include apps from all workspaces** — when that popup opens, it lists windows from every workspace, not just the current one. On by default; turn off if you only want windows from your current workspace suggested.

**Tile Groups**
- **Raise together** — when windows are tiled together as a group and you click one to bring it to front, the others in that group come to front too instead of getting buried behind other windows.

**Gaps**
- **Windows** — gap (in px) between tiled windows next to each other.
- **Screen Edges** — gap (in px) between a tiled window and the edge of the screen.
- **Maximized Windows** — toggle to also apply gaps around fully maximized windows (off by default).

#### Keybindings tab

**Dynamic Keybinding Behavior** — changes what your maximize/tile-direction keyboard shortcuts do based on a window's current tiling state:
- **Disabled** — shortcuts always do the same fixed action (default)
- **Window Focus** — pressing a direction shortcut switches focus to whatever window is already tiled in that direction, instead of moving the current window
- **Tiling State** — adapts the tiling action to context — e.g. a window already tiled to the right will tile to bottom-right if you press "tile to bottom"
- **Tiling State (Windows)** — same as above, but mimics Windows' snap behavior more closely: pressing "tile to bottom" on a window that's already at the bottom (or untiled) minimizes it instead

**Active Window Hint** — visually marks which window is currently focused:
- **Disabled** — no indicator (default)
- **Minimal** — briefly highlights the focused window only when switching to a workspace that has multiple non-overlapping windows
- **Always** — always highlights the focused window, except when maximized/fullscreen (has known display glitches with GTK4 popups on Wayland)

**Other**
- **Monitor Switch Grace Period** — when dragging a window to a different monitor to tile it, the tiling preview briefly "sticks" to the original monitor for a moment. This makes it easier to fling a window toward a screen edge even if another monitor sits right at that border. On by default.

## Notes
- Check if the extension is enabled at any time:
  ```bash
  gnome-extensions list --enabled
  ```
- If the hover popup doesn't appear after enabling, log out and log back in — Wayland sometimes needs a fresh session for shell extensions to fully apply.
- Tiling Assistant's settings panel also has options for tiling gaps, keyboard shortcuts, and "tile groups" (resizing tiled windows together) — worth exploring once the basic hover behavior is working.
