# Ubuntu/Windows Style Dock Setup on Pop!_OS (GNOME on Wayland)

## Context
- OS: Pop!_OS 24.04 LTS (fresh install)
- Session: GNOME on Wayland
- Goal: Get a proper Ubuntu-style dock

## Before you start
There are two different ways to get a "dock" look on GNOME:

- **Option A — Ubuntu Dock**: a separate floating dock (usually on the left) alongside the normal GNOME top bar. This is the classic Ubuntu look.
- **Option B — Dash to Panel**: replaces the top bar with one combined panel (clock, tray, app icons, taskbar all in one strip), which can be placed on the left to look dock-like.

**Pick only ONE. Never have both enabled at the same time** — they conflict and cause duplicate icons, broken positioning, or a frozen shell. Fully disable one before enabling the other.

Each option below is written as a complete, standalone guide — follow only the one you choose, start to finish.

---

## OPTION A — Ubuntu Dock (top bar + separate left dock)

---

### Step 1: Install Extension Manager
This is the GUI app for installing/toggling GNOME Shell extensions.

**Terminal:**
```bash
sudo apt install gnome-shell-extension-manager
```
**GUI:** Open **Pop!_Shop** → search "Extension Manager" → Install.

### Step 2: Install Ubuntu Dock
**Terminal:**
```bash
sudo apt install gnome-shell-extension-ubuntu-dock
```
**GUI (alternative):** Open **Extension Manager** → go to the **Browse** tab → search "Ubuntu Dock" or "Dash to Dock" → click **Install**.

> Note: the package `gnome-shell-extension-dashtodock` does not exist on Pop!_OS — use `gnome-shell-extension-ubuntu-dock` instead.

### Step 3: Enable the extension
**Terminal:**
```bash
gnome-extensions enable ubuntu-dock@ubuntu.com
```
**GUI:** Open **Extension Manager** → **Installed** tab → find **Ubuntu Dock** → toggle it **ON**.

### Step 4: Log out and log back in
Wayland does not support the old `Alt+F2 → r` shell-restart trick. A full logout/login is required for the extension to load properly.

### Step 5: Move the dock to the left
**Terminal:**
```bash
gsettings set org.gnome.shell.extensions.dash-to-dock dock-position 'LEFT'
```
**GUI:** Open **Extension Manager** → **Ubuntu Dock** → click the ⚙️ icon → set **Position** to **Left**.

### Step 6: Configure auto-hide
**Terminal:**
```bash
# Always visible (no auto-hide)
gsettings set org.gnome.shell.extensions.dash-to-dock autohide false
gsettings set org.gnome.shell.extensions.dash-to-dock intellihide false

# Restore default auto-hide behavior
gsettings reset org.gnome.shell.extensions.dash-to-dock autohide
gsettings reset org.gnome.shell.extensions.dash-to-dock intellihide
```
**GUI:** Open **Extension Manager** → **Ubuntu Dock** → ⚙️ → toggle **Auto-hide** on/off.

## OPTION B — Dash to Panel (single combined left panel)

### Step 1: Install Extension Manager
**Terminal:**
```bash
sudo apt install gnome-shell-extension-manager
```
**GUI:** Open **Pop!_Shop** → search "Extension Manager" → Install.

### Step 2: Install Dash to Panel
**Terminal:**
```bash
sudo apt install gnome-shell-extension-dash-to-panel
```
**GUI (alternative):** Open **Extension Manager** → **Browse** tab → search "Dash to Panel" → click **Install**.

> If the apt package isn't found on your Pop!_OS version, just use the GUI install method above — it works regardless of distro packaging.

### Step 3: Enable the extension
**Terminal:**
```bash
gnome-extensions enable dash-to-panel@jderose9.github.com
```
**GUI:** Open **Extension Manager** → **Installed** tab → find **Dash to Panel** → toggle it **ON**.

### Step 4: Log out and log back in
Required on Wayland for the extension to apply cleanly.

### Step 5: Open Dash to Panel settings
**GUI only** (Dash to Panel has no simple gsettings equivalent for most options — use its built-in settings panel):
Open **Extension Manager** → **Dash to Panel** → click the ⚙️ icon. This opens the Dash to Panel preferences window with tabs: **Position, Style, Behavior, Action, Fine-Tune, About**.

### Step 6: Move the panel to the left
In the **Position** tab → set **Panel monitor position** to **Left**.

### Step 7: Adjust thickness for a dock-like size
In the **Position** tab → set **Panel thickness** to roughly **64–96 px** so icons look more like a dock than a thin taskbar.

### Step 8: Hide non-dock elements (optional)
Still in **Position** tab, scroll the element list (Show Applications button, Activities button, Date menu, System menu, etc.) and set anything you don't want to **Hidden**, keeping just the **Taskbar** visible if you want a clean icon-only dock look.

### Step 9: Configure auto-hide
Go to the **Behavior** tab → enable/configure **Intelligent autohide** as desired.

---

## Notes
- Check which extensions are currently enabled at any time:
  ```bash
  gnome-extensions list --enabled
  ```
- To disable an extension regardless of which option you used:
  ```bash
  gnome-extensions disable <extension-id>
  ```
  (`ubuntu-dock@ubuntu.com` or `dash-to-panel@jderose9.github.com`)
- If anything looks broken after switching options, log out and back in — Wayland needs a fresh session to fully reset shell state.
