## Filesystem Types Summary

| Filesystem | Where        | Type   | Purpose                              |
|------------|--------------|--------|--------------------------------------|
| FAT32      | p1, p5, p6   | Real   | EFI boot partitions                  |
| NTFS       | p3           | Real   | Windows OS & files                   |
| ext4       | p7           | Real   | Linux OS & your files                |
| tmpfs      | /tmp, /run   | Virtual| RAM-backed, lost on reboot           |
| procfs     | /proc        | Virtual| Live kernel & process info           |
| sysfs      | /sys         | Virtual| Hardware & driver info               |
| devtmpfs   | /dev         | Virtual| Device file representations          |

## Key OS Concepts Illustrated Here

- **Everything is a file** — hardware (/dev), processes (/proc), settings (/etc) are all files
- **Virtual filesystems** — /proc, /sys, /dev don't exist on disk; kernel generates them live
- **Mount points** — partitions aren't drive letters; they're folders grafted onto the tree
- **Separation of concerns** — /bin (run), /etc (configure), /var (runtime data), /home (user data)
- **Permissions** — /root, /etc/shadow, /boot are root-only; /tmp is world-writable
