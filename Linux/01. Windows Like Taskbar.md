# COSMIC Taskbar Setup (Windows-style Single Panel)

## Context
Pop!_OS uses the **COSMIC** desktop environment (written in Rust), not GNOME.
GNOME Shell extensions like **Dash to Panel will NOT work on COSMIC** — they require GNOME Shell which COSMIC doesn't run.

The good news: COSMIC has this built-in natively.

---

## The Problem
By default, COSMIC shows:
- Pinned app launchers in the dock **+**
- A separate indicator for the running instance

This creates a "doubled" appearance unlike Windows 11 where they merge into one.

---

## The Fix (Native COSMIC Settings)

**Settings → Desktop → Panel & Dock**

1. **Delete/Disable Dock**
2. **Configure the Bottom Panel**
   - **Position:** Bottom
   - **Size:** Expand to edges (full-width, not floating macOS-style)
   - **Applets:** Drag in clock, system indicators, workspace switcher, app launcher/task list — all in one panel

---

## Notes
- ✅ Survives reboots, logouts, and system updates (it's baked into COSMIC core)
- ❌ Do NOT try `gnome-extensions` CLI on COSMIC/Wayland — it won't connect to GNOME Shell
- ❌ `gnome-shell-extension-manager` will show all extensions as "unsupported" on COSMIC

---

## What NOT to Do (Wasted Attempts)
```bash
# These do NOT work on COSMIC
git clone https://github.com/home-sweet-gnome/dash-to-panel.git \
  ~/.local/share/gnome-shell/extensions/dash-to-panel@jderose9.github.com

gnome-extensions enable dash-to-panel@jderose9.github.com  # Fails on Wayland/COSMIC
```

If you ever accidentally clone it, clean up with:
```bash
rm -rf ~/.local/share/gnome-shell/extensions/dash-to-panel@jderose9.github.com
```
