# Brave Browser — Generic Profile Shortcut Guide (Pop!_OS)

A reusable guide for creating a shortcut to launch any Brave profile directly, skipping the profile picker.

---

## Step 1 — Find the profile's directory name

1. Open the profile in Brave
2. Go to `brave://version`
3. Look at **Profile Path** — copy the last folder name (e.g. `Profile 3`, `Profile 9`, `Default`)

---

## Step 2 — Create the .desktop file

```bash
nano ~/.local/share/applications/brave-CUSTOMNAME.desktop
```

Replace `CUSTOMNAME` with a short identifier (no spaces), e.g. `brave-work`.

---

## Step 3 — Paste this content

```
[Desktop Entry]
Version=1.0
Name=Brave (YOUR_LABEL)
GenericName=Web Browser
Exec=/usr/bin/brave-browser --profile-directory="Profile X" %U
Icon=/usr/share/icons/hicolor/256x256/apps/brave-browser.png
Terminal=false
Type=Application
Categories=Network;WebBrowser;
StartupWMClass=brave-browser
```

Replace:
- `YOUR_LABEL` → display name shown in app launcher (e.g. `Work`, `Personal`)
- `Profile X` → the profile directory name from Step 1

Save: **Ctrl+O → Enter → Ctrl+X**

---

## Step 4 — Make it work

```bash
chmod +x ~/.local/share/applications/brave-CUSTOMNAME.desktop
gio set ~/.local/share/applications/brave-CUSTOMNAME.desktop metadata::trusted true
update-desktop-database ~/.local/share/applications/
```

---

## Step 5 — Launch it

**App launcher (recommended):** Press Super key → search `Brave (YOUR_LABEL)` → click

**Terminal:**
```bash
gtk-launch brave-CUSTOMNAME
```

---

## Delete a shortcut

```bash
rm ~/.local/share/applications/brave-CUSTOMNAME.desktop
update-desktop-database ~/.local/share/applications/
```

---

## List all your Brave shortcuts

```bash
ls ~/.local/share/applications/ | grep brave
```

---

## Note on pinning to dock

Pinning these custom shortcuts directly to the dock may show a generic gear icon and fail to launch (a GNOME quirk with custom `.desktop` files). The app launcher search method works reliably regardless.
