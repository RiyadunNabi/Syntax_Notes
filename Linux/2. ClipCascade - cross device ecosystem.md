# ClipCascade Setup Guide
### Seamless Clipboard Sync Between Linux (Pop!_OS) and Android

> **What is ClipCascade?**  
> A free, open-source tool that automatically syncs your clipboard (text, images, files) between your Linux PC and Android device — no button press required. The closest Linux alternative to Samsung Flow.

---

## Prerequisites

- Linux PC (Pop!_OS / Ubuntu-based)
- Android device (any)
- Both devices on the **same Wi-Fi network**
- USB cable (for one-time ADB setup)

---

## Part 1 — Install Java on Your PC

ClipCascade's server runs on Java 21. Check if it's already installed:

```bash
java -version
```

If not found, install it:

```bash
sudo apt install openjdk-21-jre-headless
```

Verify after installation:

```bash
java -version
# Should show: openjdk version "21.x.x"
```

**Disk space used:** ~203 MB

---

## Part 2 — Download and Run the Server

Create a folder for ClipCascade:

```bash
mkdir ~/ClipCascade
cd ~/ClipCascade
```

Download the server JAR:

```bash
wget https://github.com/Sathvik-Rao/ClipCascade/releases/download/3.2.0/ClipCascade-Server-JRE_21.jar
```

> Or download manually from: https://github.com/Sathvik-Rao/ClipCascade/releases  
> Look for `ClipCascade-Server-JRE_21.jar` and move it to `~/ClipCascade/`

Run the server:

```bash
java -jar ClipCascade-Server-JRE_21.jar
```

You should see Spring Boot startup logs ending with "Started ClipCascadeApplication". The server runs on **port 8080**.

---

## Part 3 — Create Your Account

Find your PC's local IP address (open a new terminal tab, keep the server running):

```bash
hostname -I
# Example output: 192.168.0.196
```

Open your browser and go to:

```
http://YOUR_IP:8080
```

Register an account (e.g., username: `admin`, password: `admin123`).  
This is local only — your data never leaves your network.

---

## Part 4 — Install the Android App

1. On your Android device, go to: https://github.com/Sathvik-Rao/ClipCascade/releases
2. Download `ClipCascade.apk`
3. Allow installation from unknown sources when prompted
4. Install and open the app
5. Enter:
   - **Server URL:** `http://YOUR_IP:8080`
   - **Username:** your username
   - **Password:** your password
   - **Disable encryption** (keep it consistent across all devices)
6. Tap **Start**

The app should show **"Connected"**.

> **Important:** Tap "Battery Optimization Settings" and disable battery optimization for ClipCascade. Otherwise Samsung/Android will kill it in the background.

---

## Part 5 — Install the Linux Desktop Client

The Linux client monitors your PC clipboard and syncs it to the server.

Download and extract:

```bash
cd ~/ClipCascade
wget https://github.com/Sathvik-Rao/ClipCascade/releases/download/3.2.0/ClipCascade_Linux.zip
unzip ClipCascade_Linux.zip
cd ClipCascade
```

Install required system dependency:

```bash
sudo apt install xclip
```

Install Python dependencies:

```bash
pip install -r requirements.txt --break-system-packages
```

Run the Linux client:

```bash
python3 main.py
```

A GUI login window will appear. Fill in:
- **Username:** your username
- **Password:** your password
- **Server URL:** `http://YOUR_IP:8080`
- **Uncheck "Enable Encryption"**

Click **Login**. The app will minimize to your system tray.

---

## Part 6 — ADB Setup for Automatic Android Clipboard Monitoring

> Android 10+ restricts background clipboard access for privacy. Without this step, syncing from Android→PC requires the app to be open. This step makes it fully automatic.

Connect your Android device via USB cable.

Enable **Developer Options** on your device:
- Go to **Settings → About Device → Software Information**
- Tap **Build Number** 7 times
- Go back to **Settings → Developer Options**
- Enable **USB Debugging**
- Accept the popup on your device

Install ADB on your PC:

```bash
sudo apt install adb
```

Verify your device is detected:

```bash
adb devices
# Should show your device serial number
```

Run these 3 commands one by one:

```bash
# 1. Grant log reading permission
adb -d shell pm grant com.clipcascade android.permission.READ_LOGS

# 2. Allow drawing over other apps
adb -d shell appops set com.clipcascade SYSTEM_ALERT_WINDOW allow

# 3. Restart the app for permissions to take effect
adb -d shell am force-stop com.clipcascade
```

Reopen ClipCascade on your Android device and tap **Start**.

---

## Part 7 — Test It

1. Copy any text on your **PC** → paste on your **Android device** ✅
2. Copy any text on your **Android device** → paste on your **PC** ✅
3. Copy an image on your **PC** → paste on your **Android device** ✅

Both directions should work instantly with no button press.

---

## Part 8 — Auto-Start on Boot (Systemd)

So you don't have to manually start the server every time.

Create a systemd service:

```bash
sudo nano /etc/systemd/system/clipcascade.service
```

Paste this (replace `YOUR_USERNAME` with your actual Linux username):

```ini
[Unit]
Description=ClipCascade Server
After=network.target

[Service]
ExecStart=java -jar /home/YOUR_USERNAME/ClipCascade/ClipCascade-Server-JRE_21.jar
Restart=always
User=YOUR_USERNAME
WorkingDirectory=/home/YOUR_USERNAME/ClipCascade

[Install]
WantedBy=multi-user.target
```

Enable and start it:

```bash
sudo systemctl daemon-reload
sudo systemctl enable clipcascade
sudo systemctl start clipcascade
```

Check status:

```bash
sudo systemctl status clipcascade
```

> For the Linux desktop client, you can add it to **Settings → Startup Applications** so it launches automatically on login.

## Troubleshooting

| Problem | Solution |
|---|---|
| Android can't reach PC | Make sure both are on the same Wi-Fi network |
| IP changed after router restart | Run `hostname -I` again and update Server URL on Android |
| "Encryption must be enabled" error | Disable encryption on all devices (Android app settings + Linux client login) |
| `xclip` not found error | Run `sudo apt install xclip` |
| App killed in background on Android | Disable battery optimization for ClipCascade |
| Server not starting | Make sure Java 21 is installed: `java -version` |

---

## Notes

- **Total disk space:** ~500 MB (Java 203 MB + server 74 MB + Linux client dependencies ~200 MB + Android APK ~89 MB)
- **RAM usage while running:** ~50–100 MB (very lightweight)
- **Network:** Works on local Wi-Fi only — no internet required, no cloud, fully private
- **IP changes:** If your router restarts or you switch networks, update the Server URL on Android with the new IP from `hostname -I`
