# Project Specification: WayClip — Wayland Clipboard Manager

**Version:** 1.2  
**Target OS:** Pop!_OS / Ubuntu 24.04 LTS (Wayland + GNOME)  
**Author:** Riyad  
**Status:** Planning

---

## 1. Problem Statement

On Wayland + GNOME, no existing clipboard manager supports auto-paste on item selection. This is a fundamental Wayland security restriction — apps cannot control each other's input at the userspace level. Existing tools like CopyQ and Clipboard Indicator either crash, fail silently, or require manual `Ctrl+V` after selection.

**WayClip** solves this by:
- Using `wl-clipboard` (native Wayland clipboard tool) for reading/writing
- Using `ydotool` (kernel-level input simulator via `/dev/uinput`) for auto-paste
- Running as a lightweight background daemon
- Showing a fast popup UI bound to a single key (`Super+V`)

---

## 2. Goals

| Goal | Description |
|---|---|
| Clipboard history | Store last 20 recent items (text + images) |
| Pinned items | User-pinned items stored permanently, survive reboots |
| Auto-paste | Selecting an item pastes it directly into the focused app |
| Single key activation | `Super+V` opens the popup |
| Wayland native | No X11 dependencies whatsoever |
| Lightweight | No database — plain text files + image folder |

---

## 3. Non-Goals

- No Windows or macOS support
- No sync across devices
- No cloud backup
- No encryption of clipboard contents (v1)
- No browser extension

---
