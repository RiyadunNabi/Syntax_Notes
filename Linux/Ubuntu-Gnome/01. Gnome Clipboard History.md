# Gnome Clipboard — Clipboard History on GNOME Wayland

## What is it?

**Gnome Clipboard** is a GNOME Shell extension that stores your clipboard history and lets you access it via a keyboard shortcut (`Super+V` by default). It works natively on Wayland — no extra daemon needed.

---

## Installation

### Step 1 — Install the Extension

Go to the GNOME Extensions website and search for **"Gnome Clipboard"**:

```
https://extensions.gnome.org
```

Or install via the **Extension Manager** app (if installed):

```bash
sudo apt install gnome-shell-extension-manager
```

Search for `Gnome Clipboard` and click **Install**.

### Step 2 — Enable it

Open the **Extensions** app → find **Gnome Clipboard** → toggle it **ON**.

---

## Fix the Super+V Conflict

By default, GNOME binds `Super+V` to the **notification panel**. You need to free it up.

**Settings → Keyboard → View and Customize Shortcuts → System → Show notification list**

Click it and either:
- Press `Backspace` to clear the shortcut, or
- Set it to something else

Now `Super+V` is free for Gnome Clipboard.

---

## Usage

- **Open history picker:** `Super+V`
- **Click any item** to copy it back to clipboard
- Then paste normally with `Ctrl+V`

---

## Configuration

Click the ⚙️ gear icon next to the extension in the Extensions app:

| Setting | Default | Description |
|---|---|---|
| Maximum history size | 100 | How many items to remember |
| Sort history by | 0 | Sort order |
| Read clipboard by timer | Off | Poll instead of event-based |
| Timer interval | 1000ms | Poll interval if timer mode on |
| Save only pinned items | Off | Store everything vs pinned only |
| Show notifications | Off | Notify on clipboard change |
| Blacklisted App IDs | — | Apps whose clipboard to ignore |
| Shortcut | `Super+V` | Picker trigger shortcut |

---

## Notes

- Works on **GNOME on Wayland** (no xdotool or daemon required)
- Auto-paste (paste directly into focused input without `Ctrl+V`) is **not supported** on Wayland due to security restrictions — this is a Wayland limitation, not the extension's fault
- If `Super+V` still opens the notification panel after setup, double-check the keyboard shortcut conflict fix above
