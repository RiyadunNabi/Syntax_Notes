# Pop!_OS Fresh Setup Guide
> My personal reference for setting up Pop!_OS from scratch (HP Victus 15, Intel i5 13th Gen, RTX 3050)

## 1. Installing VS Code

Download and install the `.deb` package:

```bash
wget -O vscode.deb "https://code.visualstudio.com/sha/download?build=stable&os=linux-deb-x64"
sudo apt install ./vscode.deb
rm vscode.deb  # clean up after install
```

When prompted **"Add Microsoft apt repository?"** → say **Yes**.  
This lets VS Code update automatically via `apt` in the future.

Launch with:
```bash
code
```

### Sync Settings
1. Click the **Accounts icon** (bottom-left)
2. **Sign in to Sync Settings** → **Sign in with GitHub**
3. All your extensions, themes, and keybindings restore automatically.

## 3. Git Setup

### Check version / Update

```bash
git --version
```

To get the latest version:
```bash
sudo add-apt-repository ppa:git-core/ppa
sudo apt update
sudo apt upgrade git   # upgrade only git, not everything else
```

> ⚠️ Always use `sudo apt upgrade <package>` to update a specific package.  
> Bare `sudo apt upgrade` upgrades everything installed.

### Configure Identity

```bash
git config --global user.name "RiyadunNabi"
git config --global user.email "riyadunnabi@gmail.com"
```

### Set Up SSH Authentication for GitHub

Instead of typing your password every time, use SSH keys.

**Step 1 — Generate a key:**
```bash
ssh-keygen -t ed25519 -C "riyadunnabi@gmail.com"
```
Press Enter through all prompts (default location, no passphrase).

**Step 2 — Copy your public key:**
```bash
cat ~/.ssh/id_ed25519.pub
```

**Step 3 — Add to GitHub:**
- Go to **github.com/settings/keys**
- Click **New SSH key**
- Title: `Pop!_OS HP Victus`
- Key type: `Authentication Key`
- Paste the key → **Add SSH key**

**Step 4 — Test:**
```bash
ssh -T git@github.com
```

Expected output:
```
Hi RiyadunNabi! You've successfully authenticated, but GitHub does not provide shell access.
```

**What this does:**  
`ssh-keygen` creates a private key (stays on your machine) and a public key (given to GitHub).
When you `git push` or `git clone`, GitHub checks if your private key matches the public key
you registered. If it matches → you're in, no password needed.
